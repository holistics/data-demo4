# Judi.ai — Holistics Enablement Program

A role-based program to get Judi.ai's team **using** Holistics on their own loan-application data —
not a feature tour. Every lesson leads with a question the person already asks, shows it on the real
**Judi Dataset**, and ends with a task they do themselves.

## Why this program exists
The data is already modelled and governed (the `test_v3` "Judi Dataset" is relationship-wired,
grouped, and carries 34 documented metrics). The remaining gap is **adoption**: getting each person to
*open, trust, and explore* the dashboards instead of waiting for a spreadsheet or emailing the data
team. This program closes that gap, role by role.

## The assets these materials cover
- **Dataset — `test_v3` "Judi Dataset"** (Endorsed): the governed menu of everything reportable.
  Centered on **Loan Application**, with Adjudication / Credit-decision, Banking & Transactions,
  Borrower & Business (+ geography), and Page-visit funnel models. 34 ready-made metrics.
- Trial is **single-tenant** (vancity) for now; multi-tenant + row-level security is a later phase.

## Recommended rollout (4 weeks)
| Week | Audience | Session | Outcome |
|---|---|---|---|
| 1 | All trial users | **Foundations** — log in, read a dashboard, Ask AI, understand the core metrics | Everyone opens their dashboard and reads the headline KPIs with confidence |
| 2 | Customer Success | **Your client-report review** | Each CS user runs their weekly review unaided, and builds one report |
| 3 | Niina + champions | **Build it yourself** | Champions create, save, and share their own reports |
| 4 | All | **Office hours / clinic** | Unblock stragglers, collect feature requests for the data team |

## Adoption levers baked into every session
- Start from a **real question**, not the tool. ("How many applications funded last month?")
- Show **Ask AI first** — it's the no-learning-required front door.
- Teach the **ⓘ info icons** and the glossary in the first 10 minutes — kills "what does this mean?" emails.
- End with a **30-second task**, not homework.

## How to use these files
- **Trainer (Niina):** read `01` + `05`, print `02`, run the tracks in `03`/`04`, pull exercises from `06`.
- **A CS user:** open your `slides/` learning page and keep `02` beside you.
- **Everyone:** the glossary (`02`) is the one thing to read before touching a number.
