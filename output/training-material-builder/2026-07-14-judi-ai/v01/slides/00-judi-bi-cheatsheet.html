The skill ladder — low → high effort

  1. Read KPIs + ⓘ icons — trust the number, tap ⓘ for its meaning
  2. Ask AI — type a plain-English question (start here)
  3. Filter — date, and client/tenant
  4. Drill down — view the applications behind a number
  5. Cross-filter — click one chart, the page re-scopes
  6. Explore + save — build your own view privately
  7. Share — send a link or add a viewer

Headline metrics

Total Applications — count of applications in view
Funded / Agreement Signed Applications — status 500 or 404
Funded / Agreement Signed Loan Amount — $ actually booked
Funded Rate — funded ÷ all applications
Auto-Approval Rate — approved ÷ auto-decisioned
Applications (Current Decision) — use when slicing by a decision field

⚠ Two gotchas everyone must know

  • One application has many decisions. Slicing a plain count by an adjudication field (auto-approval, policy, gate) over-counts. Use Applications (Current Decision) instead.
  • Credit scores are partial. Every score average excludes no-hits (0); some are sparse (~1.2k apps). Always read the matching Hit Rate for coverage. Trial data is one tenant (vancity), and the product list still has test rows — exclude them.

Raise to the data team (not a training fix)

  • A metric or field that doesn't exist yet
  • Multi-tenant / row-level security (each client sees only their data)
  • Scheduled email/Slack delivery & alerts
  • The financial-transaction path fix (interim → pull-based)
  • "Can AI answer X?" gaps → a governed AI skill
Definitions are the governed metric definitions from the Judi Dataset. When a number looks off, check the ⓘ or the metrics glossary before assuming it's wrong.
